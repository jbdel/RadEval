This changelog documents the v2.1.0 â†’ 2.2.0 change that positions RadEval

## Purpose

RadEval already exported `make_reward_fn` as a TRL-compat wrapper, but the
surface was untested against a real `GRPOTrainer`, had a latent bug on
every multi-key-per-sample metric (F1CheXbert was the confirmed offender),
and shipped a stub `examples/trl_reward_demo.py` that never instantiated
TRL. v2.2.0 hardens the reward path, adds a runnable GRPO quickstart, and
reframes the documentation to match the tested surface.

## Decisions captured (one-liners)

- **Require `key=` on multi-key metrics** (no implicit default). The
  generic check is ~3 LOC over a F1CheXbert-only patch and avoids
  re-touching `rewards.py` for the next such metric. ROUGE and RadGraph
  also need it â€” they fail loudly with the valid-keys list printed.
- **`validate_rewards` passes `None` through unchanged.** TRL uses
  `None` to skip samples in multi-task routing; coercing it via
  `float()` would crash the training loop.
- **Use `.item()` before `float()`.** Catches 0-D and 1-element 1-D
  torch tensors without a `float()` `ValueError` on some PyTorch
  versions.
- **`warnings.warn(..., UserWarning)` on the conversational heuristic**,
  not a module-level "once-only" flag with `logging.info`. User-
  filterable via standard Python; no global state under DDP.
- **No `clip_range` argument.** `score_transform=lambda x: max(lo, min(hi, x))`
  expresses it without API-surface growth or ambiguous ordering.
- **No `default_on_error`.** Silent zero-rewards hide tokenizer /
  vocab / edge-case bugs in RL loops where the reward is the only
  error signal.
- **Paper-RadCliQ positioning**: "recommended for eval / final-tune;
  benchmark per-sample cost before using for online training." No
  invented `radcliq_v0`-style composite with internal weights.
- **Qwen/Qwen2.5-0.5B in the user-facing quickstart; tiny random model
  in the CI integration test.** The 0.5B model would risk OOM or long
  downloads on a CPU CI runner; the tiny random model (~2.4M params)
  is fast and deterministic.
